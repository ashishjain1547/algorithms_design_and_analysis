Problem

Given Q queries of type L R X, find the count of integers in range [L,R] such that their Xth bit (1-indexed) is ON from the LSB (least significant bit) side.

Input Format:

  • First line contains an integer Q, denoting the number of queries.
  • Next Q lines contains three space-separated integres L R X.

Output Format:

For every query, print the number of integers in [L,R] which satisfy the above condition.

Constraints:

1≤Q≤1051≤L≤R≤1051≤X≤20

Sample Input
1
2 6 2
Sample Output
3
Time Limit: 1
Memory Limit: 256
Source Limit:
Explanation
  • Binary representation of integers in range [2,6] are as follows:-
    • 2 : 10
    • 3 : 11
    • 4 : 100
    • 5 : 101
    • 6 : 110
  • There are three integers with X-th bit ON i.e. 2, 3 and 6.
  • Thus, required answer is 3.

📌 Mathematical Observation

Bits follow a repeating pattern.

For a fixed bit position X:

  • That bit repeats every:

    2X2^X2X

  • In every cycle of size 2X2^X2X:

    • First 2X−12^{X-1}2X−1 numbers → bit is OFF

    • Next 2X−12^{X-1}2X−1 numbers → bit is ON

So it behaves like:

Code
0000 1111 0000 1111 ...

(for X=3 example)

This periodicity is the key.

✅ 4. Optimization Insight

Instead of checking each number, we count directly using pattern math.

🎯 Core Trick

Define:

f(N,X)f(N, X)f(N,X)

= number of integers from 1 to N whose X-th bit is ON.

Then answer for range:

f(R,X)−f(L−1,X)f(R, X) - f(L-1, X)f(R,X)−f(L−1,X)

This converts range query into prefix query.

🔎 How to Compute f(N, X)

Let:

block=2Xblock = 2^Xblock=2X

Each block:

  • First half → bit OFF

  • Second half → bit ON

Let:

Code
half = 2^(X-1)

Steps:

  1. Count full cycles:

    Code
    full_cycles = N // block

    Each full cycle contributes:

    Code
    half numbers

  2. Handle remaining part:

    Code
    remainder = N % block

    If remainder < half:
    → contributes 0

    Else:
    → contributes (remainder - half + 1)

So:

f(N)=full_cycles×half+max(0,remainder−half+1)f(N) = full\_cycles \times half + max(0, remainder - half + 1)f(N)=full_cycles×half+max(0,remainder−half+1)

💡 Why This Works

Because binary bits behave like a square wave:

Code
Length = 2^X
Pattern = [0...0, 1...1]

This repeats forever.

Instead of counting numbers,
we count complete patterns + leftover.

🔁 Pattern to Remember for Future

Whenever you see:

  • Count numbers in range

  • Bit at position X

  • Large Q

Think:

Bits follow periodic pattern of size 2^X

And convert:

range=prefix(R)−prefix(L−1)range = prefix(R) - prefix(L-1)range=prefix(R)−prefix(L−1)

This pattern appears in:

  • Bit counting problems

  • Parity counting

  • XOR range problems

  • Power-of-two periodic behavior

✅ 7. Clean Python Code (After Full Explanation)

Now here is the implementation:

Python
import sys
input = sys.stdin.read
data = input().split()

q = int(data[0])
index = 1

def count_upto(N, X):
    if N <= 0:
        return 0
    
    block = 1 << X          # 2^X
    half = 1 << (X - 1)     # 2^(X-1)
    
    full_cycles = N // block
    remainder = N % block
    
    count = full_cycles * half
    
    if remainder >= half:
        count += remainder - half + 1
    
    return count

results = []

for _ in range(q):
    L = int(data[index])
    R = int(data[index + 1])
    X = int(data[index + 2])
    index += 3
    
    answer = count_upto(R, X) - count_upto(L - 1, X)
    results.append(str(answer))

print("\n".join(results))
